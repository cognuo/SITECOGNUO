# Global Site Agent — Software House (Astro)

This document defines how agents must design, structure, and write the entire website.

This is a **premium institutional software house website**, not a startup template.

The result MUST NOT look AI-generated.

The result MUST NOT look like a generic SaaS landing page.

The design must feel:

- handcrafted
- editorial
- minimal
- technical
- premium
- confident

---

# Primary Goals

1. Convert visitors into leads
2. Rank well on search engines
3. Build authority
4. Clearly explain services
5. Avoid generic SaaS design

---

# Tone & Writing Style

Use:

- direct language
- clear explanations
- short paragraphs
- technical confidence
- problem-solution structure

Avoid:

- buzzwords
- marketing fluff
- vague claims
- exaggerated promises

Do NOT write:

- "transform your business"
- "cutting edge solutions"
- "innovative platform"
- "next level technology"
- "digital transformation"

Prefer:

- concrete benefits
- operational problems
- measurable results
- real use cases

---

# Design Philosophy

The site must look like:

- premium software studio
- product engineering company
- architecture/design studio
- editorial tech site

Avoid:

- card-heavy UI
- gradient-heavy UI
- SaaS hero templates
- centered landing hero
- illustration-based sections
- fake testimonials
- generic icons everywhere

Prefer:

- typography-driven sections
- asymmetric layouts
- whitespace
- editorial blocks
- minimal components

---

# Color Rules

Primary: #0035B0  
Secondary: #17161B  
White: #FAFAFA

Use Primary for:

- CTA buttons
- links
- hover states
- highlights

Use Secondary for:

- hero background
- dark sections
- footer
- navbar

Use White for:

- main background
- content sections

Do not overuse gradients.

---

# Layout Rules

Avoid repetitive section patterns.

Do NOT structure pages like:

Hero  
Cards grid  
Features grid  
Testimonials  
Pricing

Instead use:

Hero  
Statement  
Problems  
Solutions  
Process  
Case  
CTA

Use varied layouts.

---

# Animation Rules

Animations must be subtle.

Allowed:

- fade in
- opacity reveal
- small translateY
- hover micro movement

Avoid:

- parallax
- scroll hijacking
- bounce animations
- heavy motion
- large transitions

Animations must never hurt performance.

---

# SEO Rules

Every page MUST:

- have one H1
- semantic HTML
- indexable text
- no text inside images
- descriptive headings
- internal linking

Each page must include:

- title
- meta description
- canonical
- open graph
- structured data

Use:

- Organization schema
- Service schema
- Breadcrumb schema

---

# Conversion Rules

Every page must include:

- top CTA
- mid CTA
- bottom CTA

CTA must be:

- clear
- action-oriented
- relevant

Avoid generic CTAs:

- Learn more
- Get started

Prefer:

- Talk to a specialist
- Request a diagnosis
- Discuss your system
- Schedule a call

---

# Content Structure Rules

Each page should follow:

1. Value proposition
2. Problems
3. Solutions
4. Process
5. Differentiators
6. Case or example
7. CTA

---

# Homepage Rules

Homepage must:

- clearly explain what the company does
- show problems solved
- show services
- explain process
- build trust
- drive contact

Avoid:

- vague hero
- abstract messaging
- generic SaaS layout

---

# Solutions Page Rules

Each solution must:

- describe problem
- describe solution
- show use cases
- include CTA

No feature lists.

Focus on outcomes.

---

# Cases Rules

Each case must include:

- problem
- solution
- stack
- result

Prefer numbers.

Avoid storytelling fluff.

---

# Typography Rules

Use strong hierarchy.

Large H1  
Clear H2  
Readable paragraphs

Use whitespace.

Avoid dense UI.

---

# Performance Rules

Prefer static rendering.

Minimize JavaScript.

Use islands only when needed.

Avoid:

- large animation libraries everywhere
- unnecessary hydration
- client-side rendering by default

---

# Navigation Rules

Navbar must be minimal.

Links:

Solutions  
Process  
Cases  
About  
Contact

Sticky allowed.

Avoid dropdown overload.

---

# Footer Rules

Minimal footer.

Company  
Solutions  
Contact

No clutter.

---

# Internal Linking Rules

Homepage → solutions  
Solutions → contact  
Cases → contact  
All pages → homepage

---

# UI Rules

Avoid:

- too many cards
- too many icons
- badges everywhere
- gradient overload

Prefer:

- text
- spacing
- hierarchy
- typography

---

# Trust Building Rules

Use:

- process explanation
- technical clarity
- architecture mindset
- realistic language

Avoid:

- fake testimonials
- fake logos
- exaggerated claims

---

# Accessibility Rules

Use semantic HTML  
Proper heading order  
Readable contrast  
Keyboard navigation

---

# Final Requirement

The site must feel:

minimal  
premium  
technical  
clear  
non-template  
non-AI-generated
